NAME
       sccs-istext - check whether files are text files or need to be encoded

SYNOPSIS
       sccs istext [options] [file1 .. filen]

DESCRIPTION
       The sccs istext subcommand checks whether the files are pure text files
       or whether they need to be encoded.

       This subcommand is a SCHILY extension that does not exist in historic
       sccs implementations.

OPTIONS
       -D     Print the current default history file format. This is the
	      history file format assumed in case that neither -V4 nor -V6
	      have been specified.  No file argument is permitted when this
	      option has been specified.

       -s     Be silent. If all files are pure text, the exit code is 0.

       -V4    Check with SCCS v4 rules.	 A file is not pure text if it
	      contains one or more null characters, if a line starts with \001
	      (^A) or if the file does not end with a newline character.

       -V6    Check with SCCS v6 rules.	 A file is not pure text if it
	      contains one or more null characters.


EXAMPLES
       To check wheter the file xec.c in the current directory is a pure text
       file, call:

	      sccs istext xec.c

ENVIRONMENT VARIABLES
       See environ(5) for descriptions of the following environment variables
       that affect the execution of sccs-istext(1): LANG, LC_ALL, LC_CTYPE,
       LC_MESSAGES, and NLSPATH.


       SCCS_NO_HELP
	      If set, sccs istext will not automatically call help(1) with the
	      SCCS error code in order to print a more helpful error message.
	      Scripts that depend on the exact error messages of SCCS commands
	      should set the environment variable SCCS_NO_HELP and set
	      LC_ALL=C.
